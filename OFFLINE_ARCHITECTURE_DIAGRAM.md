## Key Design Principles (Shown in Diagrams)

1. **Separation of Concerns**
   - Frontend: UI, intent input, confirmation
   - Backend: Logic, policy, routing
   - Local agent: OS execution, offline queue

2. **Always Working**
   - Online: instant Supabase + optional AI
   - Offline: queued locally, syncs when online

3. **Auditable**
   - Every action stored in DB with ID
   - Pending actions tracked in felicia_pending_devices
   - Results logged in felicia_device_audit_log

4. **Token Efficient**
   - Finance logic: 0 tokens (ledger.js pure math)
   - Optional AI: separate module (ai-helper.js) for hints
   - Device control: intent parsed by Gemini once, then deterministic

5. **Graceful Degradation**
   - Online + AI: full features
   - Online - AI: core features work
   - Offline: deterministic features work, queue for sync

## Which Path Your Transaction Takes

**Transaction Type** | **Offline Path** | **Online Path**
---|---|---
Finance add | Queue locally, return success | Insert to Supabase, optional AI categorization
Task create | Queue locally, show offline | Insert to Supabase, optional AI priority hint
Goal update | Queue locally, show offline | Insert to Supabase, optional AI analysis
Device action | Queue + require confirm | Dispatch to local agent + audit log

---

## Conclusion

This architecture ensures that:
- вң… Features work offline (no network dependency)
- вң… Features sync when online (no data loss)
- вң… AI is optional (not blocking core logic)
- вң… Local agent controls OS safely (separate service, auditable)
- вң… User always knows status (online/offline badge, sync indicator)
